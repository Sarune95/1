What is pair-programming?

Pair programing is a software development technique where to people work together. One person writes the code, while the other checks each line of the code while its being written. They switch the roles frequently to maintain a high level of collaboration and work quality. This method uses excellent communication, teamwork and sharing knowledge. It achieves better work and problem solving faster.

Advantages:
Improves the quality of the code as less mistakes would be made because the other person who reviews the code they would be able to notice them when its being written. As the person who writes the code might not notice it himself. Problem solving with two people more eficient rather than one. Knowledge sharing between two people creates bigger and better ideas. Sharing the task between each other and keep swapping places maked the work more productive. Best suited for use:

How would pair-programming work in an Agile work environment?

Pair programing is great for Agile work environment as work is divided into short cycles. Each pair can work on different combinations of work and get the work done faster. This method makes sure of high level of customer service satisfactory.

Is pair-programing compatible with Agile manifesto?

Yes it is very compatible with Agile manifesto as its very efficient in software development process and flexible.

How does this method benefit a pair programming, when one programmer is far less experienced than the other?

Asking and answering questions stimulates critical thinking and ideas in pair programming. By asking questions the more experienced developer encourages the less experienced developer to think more about the task rather than only receiving instructions. They both gain better problem solving skills.
