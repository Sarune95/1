What is a use-case

UML short for Unified Modeling Language consisting of an integrated set of diagrams. A use case explains how users interact with a product or system. Use case outlines the flow of the user inputs, establishing successful and failed paths to meething the wanted goals. It allows us to better understand what the system does, why do we get errors and how does the system perform.

Elements that may be represented in a UML use-case diagram includes such as actors that the system interacts with, the system itself, the use cases or services that the system knows hot to perform and the lines that represent relationship between these elements.


Actors

Actors can be people, other systems, time triggers and event triggers. They are typically external objects of the system that produce the data:

Must serve as sources and destinations for data.

Must be external to the system.

Use-case

A use-case is a methodology used in system analysis to identify, clarify and organize system requirements. Few examples of use case are:

An airlines online booking system.

BMI app health tracker calculator.

A food delivery service mobile app.

A ticket sales platform.

System boundary

Its a rectangle that you can draw in a use-case diagram to seperate the use cases that are internal to a system from the actors that are external to the system. The system boundaries represent the diagrams parent package and are used to visually group related elements together.

The relationship between the different elements, the differences between them and enumerate them from our system

Use cases in UML diagrams demonstrate various relationships, typically representing dependencies between them. The modeler decides how two use cases are related, which can lead to reusing existing use cases, reducing effort in system definition. This reuse through relationships extends to other UML diagrams.


Include relationship

An include relationship in a use case diagram signifies that one use case incorporates the functionality of another within its business process flow. Shown by a directed arrow with a dotted shaft, the arrowhead points to the included use case, while the base connects to the including use case. The included use case cannot function independently.

Extend relationship

An extend relationship in a use case diagram signifies that a child use case enchances the functionality of the parent use case. Shown by the arrow with dotted shaft, resembling the include relationship, the arrowhead points to the parent use case, while the base connects to the extending use case.

Generalization relationship

A generalization relationship in a use case diagram establishes a pernt and a child relationship between use cases, where the child use case enchances the underlying bussiness process of the parent.
